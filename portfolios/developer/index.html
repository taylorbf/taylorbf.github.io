Ben Taylor: Work Samples

> NexusUI

NexusUI is an open-source JavaScript toolkit of musical interface components. Web Audio developers can use NexusUI to rapidly create audio projects in the browser using NexusUI and the Web Audio API. NexusUI is a popular project, being featured on Hacker News in 2014 and being used by thousands of developers around the world. Today NexusUI sees about 50 unique users each day.

Role: I have been the primary designer, developer, and maintainer since joining the project in its early stages in 2013. My role has included the development of the initial group of components, as well as steering the project through multiple redesigns and refactors, and managing community contributions and issues on GitHub.

Technologies: NexusUI is a node package that uses Webpack and Babel to bundle it for the browser. The project has gone through several iterations of technology stacks, previously using Gulp, Browserify and other tools. In 2016 I refactored the Canvas graphics engine to use SVG.

Status: NexusUI is currently open-source and used by developers around the world.

> Grapph

Grapph is an app that helps users chart progress over time. The app offers an elegantly designed interface to chart any type of ongoing data, whether that be weight, work hours, TV hours, or something else. The simple UI lets users quickly open the app, add an entry, and leave.

Role: I was the sole graphic designer and front-end developer for this project. This was a hobby project.

Technologies: Grapph was implemented in Node using a MEAN stack with Jade as the front-end language.

Status: We developed a working prototype of this project. However, my collaborator was hired as a full-time developer before we were able to finish and publish the project.

> Create1 Remixer

For the startup Create1, I developed a working prototype of their browser-based song remixer. This tool provided an interface for users to create remixes of popular songs directly in the browser, and to publish and profit from those remixes. For this project, my role was to take existing layout designs and create a working prototype in the browser using the Web Audio API.

Role: I was the sole developer of this project. I received a series of graphic designs from the team, and implemented the interface and audio engine. I was a hired contractor for this project.

Technologies: This project was written in vanilla JavaScript, using Tone.js for web audio development. Vanilla JavaScript was chosen as a development platform in order to create a lightweight app that would not interfere with the precise timing necessary to synchronize audio threads. There was no back-end work at this stage.

Status: This is a completed project which the startup uses to demonstrate their business model.
